Step 1 — Basic Order Creation
    
curl -X POST http://54.153.75.245/orders \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: test-123" \
  -d '{"customer_id":"cust1","item_id":"item1","quantity":1}'

    
    
    Expected behavior:
        HTTP 201 Created
        JSON response containing:
        order_id
        status: "created"
    Save the returned order_id for later steps.
----------------------------------------------------------
Step 2 — Retry with Same Idempotency Key
    
    
curl -X POST http://54.153.75.245/orders \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: test-123" \
  -d '{"customer_id":"cust1","item_id":"item1","quantity":1}'
    
    Expected behavior:
        Same HTTP status code as Step 1
        Same response body
        Same order_id
        No duplicate order created
    This demonstrates idempotent behavior.
----------------------------------------------------------
Step 3 — Same Key, Different Payload (Conflict Case)
 
 
curl -X POST http://54.153.75.245/orders \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: test-123" \
  -d '{"customer_id":"cust1","item_id":"item1","quantity":5}'
 
    Expected behavior:
        HTTP 409 Conflict
        No new order created
        No database modifications
    This verifies request fingerprint validation.
----------------------------------------------------------
Step 4 — Simulated Failure After Commit
    
    
curl -X POST http://54.153.75.245/orders \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: test-fail-1" \
  -H "X-Debug-Fail-After-Commit: true" \
  -d '{"customer_id":"cust2","item_id":"item2","quantity":1}'

    Expected behavior:
        HTTP 500 or timeout
        The order and ledger write must have already been committed
----------------------------------------------------------
Step 5 — Retry After Simulated Failure
    
curl -X POST http://54.153.75.245/orders \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: test-fail-1" \
  -d '{"customer_id":"cust2","item_id":"item2","quantity":1}'


    Expected behavior:
        HTTP 201 Created
        Valid order_id
        Only one order and one ledger entry exist
----------------------------------------------------------
Step 6 — Verify Order Exists

curl http://54.153.75.245/orders/<order_id>


    Expected behavior:
        Order is retrievable
        Only one order exists for that key
